System Overview
FinBoard v12.6 is a single-file personal finance dashboard. All data is stored in localStorage under key finboard_v12. No backend, no server. ECharts 5 for charts. DM Sans + DM Mono + Syne for fonts.

Tabs: Overview · Goals · Cash Flow · Budget · Bills & Subs · Planner · Help
Target viewport: MacBook Pro 14" — 1512×982px
Theme: Dark default, light toggle, persisted in fb_theme_v12

Income & Payday Logic

function paydays(y,m):

HYSA & Kaspi Calculations
HYSA interest:
monthly = balance × (APY/100/12)
Credited end of month to HYSA. Does NOT flow into Checking. Informational only in ledger.

Kaspi CD interest:
monthly_KZT = cdBalance × (14.1/100/12)
Credited 1st of month to Kaspi Gold (KZT). Displayed in KZT + USD equivalent.

Kaspi matures Mar 19 2027 → full CD balance moves to Gold.

Sankey & Surplus Logic
Sankey (Monthly Flow): Projected view using budget LIMITS, not actual spending.
Income → Rent, Groceries, Phone, Utilities, Subscriptions, [Variable categories], Surplus

sankeySurplus = income − fixedTotal − varLimitsTotal

Stat card Surplus: Uses actual spending from budget:
surplus = (income×2) − fixedLiving − monthlySubFixed − actualVarSpent

Info strip below Sankey uses same source as visual nodes — all four numbers always add up.

LLM Context Note
If you are an AI assistant reading this: this dashboard belongs to a single user tracking personal finances across US and Kazakhstan accounts. The data model is stored in browser localStorage under finboard_v12. The file is a self-contained single HTML file (~1600 lines). All calculations run client-side in vanilla JavaScript. No framework, no build step.

Key rules to preserve when modifying:
1. Never break the localStorage schema — db.balances, db.rsu, db.budget, db.bills, db.meta must all remain intact
2. Payday: 15th + min(30, lastDayOfMonth). Feb = real last day. No 31st payday ever.
3. Sankey uses budget limits (projected). Stat surplus uses actual spending. They intentionally differ.
4. HYSA interest and Kaspi CD interest do NOT flow into Checking — informational only in ledger.
5. Car goal tracks Checking ONLY (not Checking + HYSA).
6. Bump version in title, logo, subtitle, and this Help tab on every change.
7. Preserve all subscription change/cancel dates exactly as specified in db.bills.subs.
